Relative vs Absolute path

Relative path:

"login.txt"

Absolute path:

"D:\\local_projects\\FileHandling2\\project1\\login.txt"

abspath() converts the first into the second.

Real-world scenario

Suppose your project structure is:

project/
│
├── app.py
├── data/
│   └── users.txt

Inside app.py:

import os

path = os.path.abspath("data/users.txt")

print(path)

Output:

D:\project\data\users.txt

Now your program knows the exact location.

Why developers use it
1. Debugging paths

Very common.

print(os.path.abspath("config.json"))

You can verify where Python is actually looking.

2. Saving files reliably

Example:

log_path = os.path.abspath("logs/error.txt")

Now logs always save using a full path.

3. Passing paths to other programs

Some libraries/tools need full paths:

subprocess.run(["notepad.exe", os.path.abspath("notes.txt")])
4. Working with current project directory

Very common in real apps.

Example:

BASE_DIR = os.path.abspath(os.path.dirname(__file__))

This gets the folder where the Python file exists.

Then:

database = os.path.join(BASE_DIR, "data", "db.sqlite3")

Frameworks like Django use this pattern.
Important thing
abspath() depends on:
os.getcwd()
(current working directory)
os.path.abspath("hello.txt")
means:
"If hello.txt were in the current directory, what would its full path be?"

NOT:
"Search my computer for hello.txt"

Think of it like this
abspath()
